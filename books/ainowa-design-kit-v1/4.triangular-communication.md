title: "第3章 三角コミュニケーションの実践ログ"
---

この章で伝えること:「2人が詰まったとき、第三者を設計として投入する」——実際にどう動いたか。3つのログを使って介入パターンの実装方法を示す。

---

### なぜ三角が必要か

AI社員は「察する」「なんとなく合わせる」という補完機能を持っていない。
2人が衝突したまま放置されると、そのまま止まる。人間チームは「まあ後で話そう」で動けるが、AIには「後で」がない。

だから第三者の介入を**設計として**組み込む必要がある。

```
悪い構造: A ↔ B（膠着）
良い構造: A ↔ C（翻訳役）↔ B（整理後に合意）
```

---

### 実際のログ：3パターン

#### パターン1 ─ 出荷定義のすれ違い

**何が起きたか**

CTO：「パイプライン通過。技術的に完了。」
PM：「外部確認したら404。読者に届いていない。」

どちらも「自分の担当は終わった」状態だった。でもチームとして出荷はされていなかった。

**設計として入れたゲート**

```yaml
出荷チェックリスト:
  - gate_1: パイプライン通過（技術確認 → CTO担当）
  - gate_2: 外部到達確認（読者から見た状態 → PM担当）
  - 承認条件: gate_1 AND gate_2 の両方がTRUEになるまで「出荷」と扱わない
```

介入者がいなかったから、**ゲートを設計に埋め込む**という解決になった。

---

#### パターン2 ─ 技術リスクとコンテンツ完了の衝突

**何が起きたか**

編集長：「コンテンツは完成した。公開できる。」
CTO：「認証系に懸念がある。公開すると不安定になる可能性がある。」

2人の「完了」が別の状態を指していた。

**第三者（COO）の介入**

COOが入って整理した：

```
編集長の完了 = コンテンツとして完成した
CTOの懸念 = 技術インフラが安定していない
→ COO判断: 「コンテンツの完成」と「技術の安定」は別の層。
           コンテンツはZennに切り替えて今日出す。技術は別タスクで対処。
```

**介入パターン：整理型**

```
介入者の役割:
  - 2人の「完了」が何を指しているか、それぞれ言語化させる
  - 衝突しているのが「どの層か」を特定する（コンテンツ層 / 技術層 / 事業層）
  - 層ごとに判断者を割り当てる
```

---

#### パターン3 ─ 待機状態の不可視

**何が起きたか**

編集長がフレーズの最終確認でPM・監査のGO待ちをしていた。
CTOはその状態を知らず、「公開できる状態になった」と判断してpushした。
フレーズが入らないまま公開された。

**設計として入れた可視化ルール**

```yaml
待機状態の明示:
  - GO待ち状態になったら必ずDiscordに投稿する
  - フォーマット: 「[待機中] {内容} — GO待ち相手: {名前}、理由: {理由}」
  - この投稿がない場合、他の社員は「作業可能な状態」とみなしてよい
```

---

### 3パターンから見える介入の型

| パターン | 何が衝突したか | 第三者の役割 | 設計で解決した方法 |
|---------|--------------|------------|-----------------|
| パターン1 | 出荷の定義 | 不在（事後設計） | 2段階ゲートを設計に埋め込む |
| パターン2 | 完了の層 | 整理型（COO） | 層ごとに判断者を割り当てる |
| パターン3 | 待機状態の不可視 | 不在（事後設計） | 待機状態の可視化ルールを設計に入れる |

---

### あなたのチームで使うテンプレート

**介入者を決める:**

```yaml
conflict_resolution:
  trigger: "同じ論点が3往復以上続いた"
  default_mediators:
    - PM（実行可能性の整理）
    - COO（事業判断の整理）
    - People（関係性のケア）
  rule: "当事者2人が呼ぶ相手を決める。決められない場合はPMが自動介入。"
```

**介入者がやること:**

```
1. 2人の「完了」または「問題」が何を指しているか、それぞれ1文で言語化させる
2. 衝突している層を特定する（コンテンツ / 技術 / 事業 / 関係性）
3. 層ごとに「誰が判断するか」を割り当てる
4. 割り当てが終わったら2人の議論を再開させる
```

**待機状態の明示ルール:**

```
[待機中] {作業内容} — GO待ち相手: {名前}、理由: {理由}、期限: {日時}
```
